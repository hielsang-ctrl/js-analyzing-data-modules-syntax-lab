# Contributing Guide

## Version Control

- Use Git to track every change to lab instructions, solution code, and tests.
- Create short-lived branches for each update, bug fix, or documentation change.
- Write focused commit messages that explain the intent of the change.
- Open a pull request before merging work into the main branch when collaborating.

## Reviews And Updates

- Re-run `npm test` before opening or merging a change.
- Review this lab regularly to keep examples aligned with current JavaScript syntax and tooling.
- Check that instructions, starter code, and tests still agree with each other after every update.
- Record meaningful maintenance changes in pull request descriptions so future contributors have context.

## Documentation Repository

- Keep lab instructions in `README.md`.
- Keep implementation code in `index.js`.
- Keep automated checks in `test/index.test.js`.
- Keep project-level maintenance guidance in `CONTRIBUTING.md`.
- Keep documentation indexes or supporting notes in the `docs/` directory.
